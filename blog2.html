Challenge #2: Training Dataset and Quality Assurance

Anyone who knows data science would agree with me that data collection is the most important and challenging part in the AI/ML project.
It is also important to consider and a collect dataset with meaningful keywords, define those properly inside the system, which scenarios they could emerge, which components do they affect, serverity, linked vulnerabilities, code based analysis, malicious behaviours, behavioural analysis, reproduction steps, scripts and code snippets, vulnerability name, runtime anomolies, dynamic and static behaviours, text based description(unstructured documentation), keywords and their priorities, analysis on kernel, static, dynamic, runtime, rest and diffent levels, scoring, decision making and false positives detection.
I decided that I would spend months preparing for the dataset, worst case I would spend 25% of the planned project effort, but would not to compromise on the traing data.
