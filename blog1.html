Challenge #1: Thoughts on which Attributes to select for training

To select appropriate attributes that can provide the best results are key when building an AI-based security testing model.

Two things, AI tool should be able to do both static analysis and dynamic analysis. Static analysis would involve a specific piece of code that is malicious in nature, leads to security vulnerabilities, some malware is present inside the code, etc. The AI tool should be able to comprehensively detect these things. While in dynamic analysis, the tasks are more challenging since the AI model should be able to detect events, patterns in the app, network, kernel, bridge, protocol behaviors to comprehensively detect vulnerabilities.

I would use my step by step process. Analyzed what technologies do present AI-based tools use to effectively find vulnerabilities. Plan to use NLP Models(eg BERT, GPT), try pretrained models like Hugging Face's Transformers. Also use of Rule-Based Systems(that don't require Machine Learning) which can be easily integrated.

Later, I changed the initial idea and planned to consider numbers based on their privileges and ranks. However, some people suggested that it is not a good idea, instead, switch back to text-based attributes.
I understood that attributes should be text-based for better results. So now, should I consider text-based lines or some keywords which indicate what actions happened in the system?
Finally, I decided that for better results, I should consider text-based remediation and other attributes could be keywords that indicate some sort of event that happened on the system.
For each keyword, some kind of software program or automation should be written to know what the attribute refers to. Of course, some attributes could be common between activities that would show that some kind of relationship is there between attributes.
Most importantly, based on these attributes, the model should be able to understand what kind of event is running on the mobile app and decide what kind of testing should be done on these.
